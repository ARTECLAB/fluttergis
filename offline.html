El problema del campo
Por qué offline-first no es opcional en GIS móvil
📡
Realidad del trabajo de campo
En municipios rurales de Bolivia la cobertura 4G es irregular o inexistente. Una app que depende de internet para mostrar el mapa es inútil en estas condiciones.
✓
Modo avión GIS completo
Tiles del mapa base desde FMTC cache. GeoJSON desde SQLite. GPS del dispositivo. Colectas guardadas local. La app es 100% operativa sin señal.

Setup FMTC
Inicializar el store y conectarlo al mapa
main() + TileLayer con FMTC
void main() async { WidgetsFlutterBinding.ensureInitialized(); // Inicializa el backend de FMTC (ObjectBox) await FMTCObjectBoxBackend().initialise(); // Crea el store si no existe todavía await const FMTCStore('descargas') .manage.create(); runApp(const GeoCollectApp()); } // tileProvider del TileLayer del mapa: TileLayer( urlTemplate: 'https://tile.openstreetmap.org' '/{z}/{x}/{y}.png', tileProvider: FMTCTileProvider( stores: { // read: SOLO lee el store, nunca // escribe -- navegar no cachea nada 'descargas': BrowseStoreStrategy.read, }, loadingStrategy: BrowseLoadingStrategy.cacheFirst, ), userAgentPackageName: 'com.arteclab.geocollect', )
✓
BrowseLoadingStrategy.cacheFirst
FMTC intenta primero el cache local, y si no existe el tile, lo descarga de la red. En campo sin internet, solo usa lo que ya tiene.
🤔
Por qué read y no readUpdateCreate?
FMTC también permite cachear automáticamente todo lo que navegas. Pero eso crece sin control y no garantiza haber recorrido TODA tu zona de trabajo antes de perder señal. Con read, lo único que llena el cache es el botón de descarga: predecible, "lo que descargaste es lo que tienes".
⚠️
La API cambió en FMTC v10: ya no existe FlutterMapTileCaching.initialise() ni FMTCTileProviderSettings. Se inicializa con FMTCObjectBoxBackend() y el tile provider se construye con FMTCTileProvider directamente.

// Se captura el área que el mapa muestra // en este momento, no una región fija final bounds = mapController.camera.visibleBounds; final zoomActual = mapController.camera.zoom; final z = zoomActual.round(); final region = RectangleRegion(bounds) .toDownloadable( // Un nivel más alejado: no perder // tiles con un paneo chico minZoom: (z - 1).clamp(1, 19), // 3 niveles más cercanos: poder // acercarse sin señal al punto exacto maxZoom: (z + 3).clamp(1, 19), options: TileLayer( urlTemplate: 'https://tile.openstreetmap.org' '/{z}/{x}/{y}.png', userAgentPackageName: 'com.arteclab.geocollect', ), ); // Contar tiles y estimar tamaño ANTES // de descargar -- nunca a ciegas final tiles = await const FMTCStore( 'descargas').download.countTiles(region); final estMB = tiles * 3 / 1024; // ~3KB/tile
🗺️
Cómo las apps de mapas offline reales
Google Maps, Organic Maps: navegas hasta la zona que te interesa y descargas exactamente eso — no una región que alguien más eligió por ti de antemano.
⚠️
Si la estimación supera ~2000 tiles, hay que avisar antes de descargar: puede tardar varios minutos, o el servidor gratuito puede cortar la descarga a la mitad (siguiente slide).
📐
countTiles() reemplaza a check(), deprecado en FMTC v10.1 — mismo resultado, nombre más claro.
Hack bueno de la clase · Parte 2 de 2
tile.openstreetmap.org no es para descarga masiva
Descargar siendo buen ciudadano del server
// Más lento que el default, pero más // respetuoso del servidor gratuito final download = const FMTCStore( 'descargas').download.startForeground( region: region, parallelThreads: 2, // default: 5 rateLimit: 10, // tiles/segundo máx ); DownloadProgress? _ultimoProgreso; download.downloadProgress.listen((p) { setState(() { _progreso = p.percentageProgress; _ultimoProgreso = p; // para el resumen }); }, onDone: () { // El stream se cierra IGUAL si terminó // solo o si se canceló -- por eso hay // que comparar tiles reales vs esperados final p = _ultimoProgreso!; final completo = p.successfulTilesCount >= p.maxTilesCount; // Mostrar SnackBar: completo o // incompleto, con los números reales }); // download.pause() / .resume() existen // si se corta el WiFi a medio camino
🚫
La política de uso de OSM lo prohíbe
tile.openstreetmap.org es gratuito y público — su política de uso prohíbe expresamente la descarga masiva ("bulk downloading"). Con miles de tiles seguidos, el servidor empieza a rechazar peticiones a la mitad. flutter_map hasta imprime una advertencia sobre esto al iniciar la app.
🪪
El userAgentPackageName también va en la descarga masiva, no solo en el mapa en vivo — OSM exige un User-Agent identificable, sin él es más fácil que el servidor rechace las peticiones.
✓
"Terminó" no siempre significa "se descargó todo"
Comparar successfulTilesCount con maxTilesCount es la única forma de confirmar si la descarga realmente se completó, o si se cortó a medias sin que se note.
Verificación
Cómo saber que el mapa SÍ está usando lo descargado
Estadísticas reales del store
// Cuántos tiles hay, cuánto pesan, y // cuántas veces se usaron de verdad Future<({int tiles, double sizeMB, int hits, int misses})> estadisticas() async { final s = await const FMTCStore( 'descargas').stats.all; return ( tiles: s.length, sizeMB: s.size / 1024, hits: s.hits, // servido del cache misses: s.misses, // pedido a la red ); }
👀
Un tile de cache y uno de red se ven IGUAL
No hay ninguna señal visual que diga de dónde vino un tile. hits subiendo mientras navegas dentro del área descargada es la única forma de comprobar que el cache realmente se está usando.
📍
Cuidado al probar modo avión
La Clase 2 hace que el mapa siga tu GPS real en cada actualización. Si al probar te alejas del área que descargaste (o tu GPS real está en otro lado), el mapa se recentra fuera del cache — y ahí sí, sin red, no hay tile que mostrar. Apagar la Ubicación antes de activar modo avión evita esto.
✓
La prueba definitiva sigue siendo modo avión real, pero el panel de estadísticas permite comprobarlo en vivo, en clase, sin depender del WiFi del aula.
Resumen
Lo que aprendiste hoy
  • FMTC con BrowseStoreStrategy.read: navegar el mapa NO cachea nada por su cuenta — solo el botón de descarga llena el store
  • La región a descargar se arma con el área VISIBLE del mapa (visibleBounds + zoom actual), no un BBox fijo — igual que Google Maps u Organic Maps
  • countTiles() (no check(), deprecado) estima tiles antes de descargar; el tamaño se calcula multiplicando por un promedio de KB por tile
  • tile.openstreetmap.org prohíbe la descarga masiva — parallelThreads bajo, rateLimit y userAgentPackageName reducen el riesgo de que el servidor corte la descarga
  • El stream de progreso se cierra igual si la descarga termina bien o se cancela — comparar successfulTilesCount vs maxTilesCount confirma si de verdad se completó
  • Un tile de cache y uno de red se ven igual: hits/misses vía FMTCStore.stats es la única forma de comprobar que el mapa usa lo descargado, sin cortar la señal
