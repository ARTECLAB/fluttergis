El problema del campo

📡
Realidad del trabajo de campo
En municipios rurales de Bolivia la cobertura 4G es irregular o inexistente. Una app que depende de internet para mostrar el mapa es inutil en estas condiciones.
✓
Modo avion GIS completo
Tiles del mapa base desde FMTC cache. Capas WFS desde SQLite. GPS del dispositivo. Colectas guardadas local. La app es 100% operativa sin señal.
Tiles base (FMTC)OSM, CartoDB — descargados antes del campo
Capas vectorialesSQLite cache con TTL (Clase 4)
GPSFunciona sin internet — señal satelite
ColectasSQLite local, sincroniza al volver

✓
CacheBehavior.cacheFirst
FMTC intenta primero el cache local, y si no existe el tile, lo descarga de la red. En campo sin internet, solo usa lo que ya tiene.
📦
FMTC organiza los tiles en stores — como carpetas. Puedes tener un store para la ciudad y otro para el area rural, y cambiar entre ellos.

📊
Estimacion tipica La Paz
Zoom 10-16: ~15,000 tiles, ~45MB. Zoom 10-14: ~1,000 tiles, ~3MB. Mostrar siempre la estimacion antes de descargar.
⚠️
Descargar sin estimacion previa puede llenar el almacenamiento del dispositivo. Con BBox grande y zoom 17+, puede ser varios GB.
Conectividad automatica
Switch online/offline transparente con connectivity_plus
Cambio automatico de modo
class _MapState extends ConsumerState<MapScreen> { var _esOnline = true; @override void initState() { super.initState(); // Escuchar cambios de red Connectivity().onConnectivityChanged .listen((result) { final online = result != ConnectivityResult.none; if (online != _esOnline) { setState(() => _esOnline = online); if (online) { // Al reconectar: sincronizar ref.read(colectasProvider.notifier) .sincronizar(); } } }); } // El TileLayer cambia segun conectividad: // Online: descarga tiles nuevos + cachea // Offline: usa solo FMTC cache // El codigo del TileLayer es identico // FMTC maneja todo transparentemente }
✓
FMTC maneja el cambio online/offline transparentemente — el TileLayer es identico en ambos modos. La app no necesita codigo adicional para el cambio.
⚡
Al reconectar, lanzar la sincronizacion de colectas pendientes — el ciclo offline-first completo: campo sin señal, regreso con datos, sincronizacion automatica.
Resumen
Lo que aprendiste hoy
  • FMTC organiza tiles en stores — crear el store antes de descargar o cachear
  • CacheBehavior.cacheFirst: usa cache si existe, descarga de red si no — transparente para la app
  • Estimar el tamano de la descarga con check() antes de proceder — evita llenar el storage
  • La Paz zoom 10-16: ~15,000 tiles, ~45MB — descargable en WiFi antes del campo
  • connectivity_plus.onConnectivityChanged dispara la sincronizacion de colectas al reconectar
  • Modo avion GIS: FMTC + SQLite + GPS = app 100% funcional sin señal de celular
