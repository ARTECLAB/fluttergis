los datos que viajan

GeoJSON en Dart puro · CSV en Dart puro · share_plus · geojson.io

⏱ 2 horas📤 Export👨‍💻 Daniel Quisbert
Por qué exportar
Los datos que se quedan solo en el celular no sirven a nadie
🤔
El problema que resolvemos hoy
Hasta la Clase 8, las colectas de campo viven solo en el SQLite del celular que las capturó. Si ese celular se pierde, se rompe o simplemente lo usa otra persona, los datos no existen para nadie más. Exportar es la forma de sacar esos datos del celular y ponerlos en manos de quien los necesita.
👥
Dos públicos, dos formatos
No todos los que reciben los datos de campo son especialistas GIS. Por eso GeoCollect exporta a dos formatos distintos, cada uno pensado para quien lo va a abrir.
🗺️
GeoJSON — para el equipo GIS
Formato estándar para datos vectoriales. Lo abren QGIS, geojson.io, Leaflet y OpenLayers sin conversión. Guarda geometría (dónde está el punto) y atributos (qué se colectó) en un solo archivo de texto.
📊
CSV — para todos los demás
Lo abre Excel, Google Sheets o cualquier responsable de proyecto sin instalar nada. La latitud y la longitud aparecen como columnas normales, junto al resto de atributos de cada colecta.
GeoJSON en Dart puro
Convertir List<Colecta> a un FeatureCollection válido
lib/services/export_service.dart — parte 1
String colectasAGeoJson( List<Colecta> colectas) { // map() transforma cada Colecta en un // Map que respeta la estructura Feature final features = colectas.map((c) => { 'type': 'Feature', 'geometry': { 'type': 'Point', // GeoJSON exige [lon, lat], NO [lat, lon] 'coordinates': [c.longitud, c.latitud], }, 'properties': { // Todo lo que no es geometría va aquí 'id': c.id, 'tipo': c.tipo, 'descripcion': c.descripcion, 'timestamp': c.timestamp, 'foto': c.fotoPath != null, }, }).toList(); // El FeatureCollection envuelve todas // las features en un solo objeto final fc = { 'type': 'FeatureCollection', 'features': features, }; // jsonEncode convierte el Map a texto return jsonEncode(fc); }
lib/services/export_service.dart — parte 2
Future<void> exportarGeoJson( List<Colecta> colectas) async { final json = colectasAGeoJson(colectas); // path_provider da una carpeta válida // tanto en Android como en iOS final dir = await getApplicationDocumentsDirectory(); final file = File('${dir.path}/colectas.geojson'); // Escritura asíncrona: no bloquea la UI await file.writeAsString(json); // share_plus abre el selector nativo // del sistema: WhatsApp, correo, Drive... await Share.shareXFiles( [XFile(file.path)], text: 'Colectas GeoCollect — ARTECLAB', ); }
⚠️
El error más frecuente del curso
Invertir el orden y escribir [c.latitud, c.longitud] coloca los puntos en el océano. La regla para recordarlo: en GeoJSON las coordenadas se leen igual que en un plano cartesiano, primero el eje X (longitud) y después el eje Y (latitud).
CSV en Dart puro
La misma lista de colectas, ahora como filas y columnas
lib/services/export_service.dart — parte 3
String colectasACsv( List<Colecta> colectas) { // StringBuffer arma el texto sin crear // un String nuevo en cada línea final buffer = StringBuffer(); // Primera fila: encabezados de columna buffer.writeln( 'id,tipo,descripcion,latitud,' 'longitud,timestamp'); for (final c in colectas) { // _csvSafe evita que una coma dentro // de la descripción rompa las columnas buffer.writeln([ c.id, c.tipo, _csvSafe(c.descripcion), c.latitud, c.longitud, c.timestamp, ].join(',')); } return buffer.toString(); } // Envuelve el texto en comillas si // contiene comas, comillas o saltos String _csvSafe(String texto) { if (texto.contains(',') || texto.contains('"')) { final escapado = texto.replaceAll('"', '""'); return '"$escapado"'; } return texto; }
✓
lat/lon como columnas normales
A diferencia de GeoJSON, en CSV no hay una regla de orden que respetar: latitud y longitud son dos columnas más, con el nombre que les pongamos en el encabezado. Excel y Sheets las grafican en un mapa con "Insertar > Mapa" sin ningún paso adicional.
💬
Prompt para la IA: "Escribe una función Dart que convierta List<Colecta> a un String CSV, escapando comas y comillas en el campo descripción." La IA suele olvidar el escape de comillas dobles — revisar esa parte con un dato de prueba que tenga comas.
⚠️
Guardar y compartir el CSV usa el mismo patrón que el GeoJSON: File(...).writeAsString(csv) seguido de Share.shareXFiles(...). Por eso conviene juntarlos en un solo método — lo vemos en la siguiente slide.
Hack
Preview en geojson.io
sin instalar nada
Un link compartible generado en Dart con base64 — se abre en cualquier navegador, sin servidor propio
Hack bueno de la clase
Un link que muestra el mapa antes de abrir QGIS
lib/services/export_service.dart — parte 4
import 'dart:convert'; // HACK: link de preview en geojson.io // sin necesitar servidor propio String generarLinkPreview( List<Colecta> colectas) { final json = colectasAGeoJson(colectas); // base64 permite meter el archivo // completo adentro de la URL final b64 = base64Encode( utf8.encode(json)); return 'https://geojson.io' '/#data=data:application/json' ';base64,$b64'; } // Comparte GeoJSON y CSV juntos con // un solo mensaje, para no elegir // entre el público GIS y el general Future<void> exportarTodo( List<Colecta> colectas) async { final dir = await getApplicationDocumentsDirectory(); final geoFile = File('${dir.path}/colectas.geojson'); await geoFile.writeAsString( colectasAGeoJson(colectas)); final csvFile = File('${dir.path}/colectas.csv'); await csvFile.writeAsString( colectasACsv(colectas)); await Share.shareXFiles( [XFile(geoFile.path), XFile(csvFile.path)], text: 'Colectas GeoCollect — ARTECLAB', ); }
✓
Por qué este hack importa
El link codifica el GeoJSON completo dentro de la URL. Quien lo recibe por WhatsApp lo abre en el navegador de su celular y ve los puntos en el mapa al instante, sin instalar QGIS ni tener acceso a ningún servidor. Funciona bien hasta archivos de unos 500KB — con más colectas que eso, conviene compartir el archivo en vez del link.
📎
Un solo botón, dos archivos
Share.shareXFiles acepta una lista, no un solo archivo. Adjuntar el GeoJSON y el CSV en la misma llamada evita que el usuario tenga que exportar dos veces y cubre a los dos públicos con una sola acción.

🔗
La regla de la arquitectura, aplicada aquí
Igual que en el resto del proyecto (Clase 7), las flechas solo bajan: ExportScreen pide los datos a ColectaDB, se los pasa a ExportService, y ExportService entrega el archivo listo para compartir. La pantalla nunca arma el GeoJSON ni el CSV por su cuenta.
🧭
De dónde salen las colectas a exportar
ExportScreen pide todas las colectas guardadas con ColectaDB.listar() — las mismas que ya se ven como Marker en el mapa desde la Clase 5. Exportar no es un módulo aislado: reúsa los datos que la app ya viene coleccionando.
Resumen
Lo que aprendiste hoy
  • GeoJSON en Dart puro: FeatureCollection con features tipo Point — coordinates: [lon, lat], nunca al revés
  • CSV en Dart puro: StringBuffer + join(',') + escape de comas y comillas en _csvSafe()
  • lon antes que lat en GeoJSON — al revés que LatLng(lat, lon) de flutter_map, por eso el error es tan común
  • share_plus.ShareXFiles comparte uno o varios archivos con WhatsApp, correo o Drive en una sola llamada
  • Link geojson.io con base64: preview inmediato compartible sin servidor propio, útil hasta ~500KB
  • ExportScreen solo llama a ExportService — la misma regla de arquitectura de siempre, sin excepciones
