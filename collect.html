El modelo de datos
La clase Colecta — toMap() y fromMap()
lib/models/colecta.dart
class Colecta { final int id; final String tipo; final String descripcion; final double latitud; final double longitud; final String? fotoPath; final int timestamp; final bool sincronizado; Map<String,dynamic> toMap() => { 'id': id, 'tipo': tipo, 'descripcion': descripcion, 'latitud': latitud, 'longitud': longitud, 'fotoPath': fotoPath, 'timestamp': timestamp, // bool no existe en SQLite 'sincronizado': sincronizado ? 1 : 0, }; factory Colecta.fromMap(Map m) => Colecta( id: m['id'], tipo: m['tipo'], descripcion: m['descripcion'], latitud: m['latitud'], longitud: m['longitud'], fotoPath: m['fotoPath'], timestamp: m['timestamp'], sincronizado: m['sincronizado'] == 1, ); }
⚠️
bool → INTEGER en SQLite
SQLite no tiene tipo BOOLEAN. La IA pasa el bool directamente y lanza error en runtime. Convertir siempre: sincronizado ? 1 : 0
✓
ID único sin servidor: DateTime.now().millisecondsSinceEpoch — garantiza unicidad sin autoincrement del servidor.
Base de datos SQLite
Inicializar, insertar y consultar colectas
lib/data/colecta_db.dart
class ColectaDB { static Database? _db; static Future<Database> get db async { _db ??= await _initDb(); return _db!; } static Future<Database> _initDb() async { final dir = await getApplicationDocumentsDirectory(); return openDatabase( join(dir.path, 'geocollect.db'), version: 1, onCreate: (db, _) => db.execute(''' CREATE TABLE colectas ( id INTEGER PRIMARY KEY, tipo TEXT NOT NULL, descripcion TEXT, latitud REAL NOT NULL, longitud REAL NOT NULL, fotoPath TEXT, timestamp INTEGER NOT NULL, sincronizado INTEGER DEFAULT 0)'''), ); } static Future<int> insertar(Colecta c) async => (await db).insert( 'colectas', c.toMap(), conflictAlgorithm: ConflictAlgorithm.replace); static Future<List<Colecta>> pendientes() async { final rows = await (await db).query( 'colectas', where: 'sincronizado = 0'); return rows.map(Colecta.fromMap).toList(); } }
📁
getApplicationDocumentsDirectory() devuelve la ruta correcta en cada plataforma — el código Dart es idéntico en Android e iOS.
✓
sincronizado = 0 en la consulta SQL filtra las colectas pendientes — base del sistema offline-first. Al sincronizar, actualizar a 1.

Formulario de colecta con GPS automático y foto comprimida
GPS automático al abrir el formulario
class _FormState extends State<FormColecta> { LatLng? _pos; String? _fotoPath; @override void initState() { super.initState(); _cargarPosicion(); // GPS al abrir } Future<void> _cargarPosicion() async { // Usar ultima posicion conocida // (instantaneo, sin esperar fix nuevo) final p = await Geolocator.getLastKnownPosition() ?? await Geolocator.getCurrentPosition(); setState(() => _pos = LatLng(p.latitude, p.longitude)); } Future<void> _tomarFoto() async { final img = await ImagePicker() .pickImage( source: ImageSource.camera, imageQuality: 30, // 3MB -> ~80KB maxWidth: 1024, ); if (img != null) setState(() => _fotoPath = img.path); } }
imageQuality: 30~80KB — detalle suficiente para documentar
imageQuality: 100~3MB — innecesario en campo
Capacidad campo1000 fotos en 80MB vs 3GB
⚡
getLastKnownPosition() es instantáneo — usa la posición en memoria. getCurrentPosition() puede tardar 5 segundos buscando fix. Siempre intentar last conocida primero.
Offline-first
Guardar local + sincronizar cuando hay internet
Ciclo offline-first completo
// Al guardar la colecta: Future<void> guardarColecta( Colecta c) async { // Siempre guardar local primero await ColectaDB.insertar(c .copyWith(sincronizado: false)); // Intentar sincronizar si hay internet final conn = await Connectivity() .checkConnectivity(); if (conn != ConnectivityResult.none) { await _sincronizar(); } } // Escuchar reconexion automaticamente void _listenConectividad() { Connectivity().onConnectivityChanged .listen((result) { if (result != ConnectivityResult.none) { _sincronizar(); } }); } Future<void> _sincronizar() async { final pendientes = await ColectaDB.pendientes(); for (final c in pendientes) { // Publicar en GeoServer via REST await _publicarEnGeoServer(c); // Marcar como sincronizado await ColectaDB.marcarSincronizado(c.id); } }
✓
Flujo offline-first
1. Guardar en SQLite → 2. Intentar sincronizar → 3. Si no hay internet: esperar. 4. Al reconectar: sincronizar automáticamente. Nunca se pierde una colecta.
📡
onConnectivityChanged es un Stream que emite cuando cambia la red — el dispositivo detecta la reconexión automáticamente y dispara la sincronización.
IA como copiloto
Generar el modelo Colecta con Claude
💬
"Genera el modelo Dart 'Colecta' con campos: id, tipo, descripcion, latitud, longitud, fotoPath, timestamp, sincronizado. Con toMap() y fromMap() para sqflite. Incluye validaciones básicas."
Lo que la IA hace bien
La estructura de la clase, los campos, el constructor, y la mayor parte de toMap()/fromMap(). Correcto al 85%.
Errores típicos de la IA
1) Pasar bool directamente a SQLite sin convertir a 0/1. 2) Olvidar el campo sincronizado en el schema SQL. 3) No marcar latitud/longitud como NOT NULL en el CREATE TABLE.
✓
Checklist después de generar
Verificar: 1) sincronizado ? 1 : 0 en toMap(), 2) m['sincronizado'] == 1 en fromMap(), 3) REAL NOT NULL en latitud y longitud del schema SQL.
Resumen
Lo que aprendiste hoy
  • SQLite no tiene BOOLEAN — siempre convertir: sincronizado ? 1 : 0. La IA olvida esto
  • toMap() para insertar en SQLite, fromMap() para reconstruir desde el resultado SQL
  • getLastKnownPosition() es instantáneo vs getCurrentPosition() que puede tardar 5 segundos
  • imageQuality: 30 comprime de 3MB a ~80KB — multiplica x37 la capacidad de almacenamiento en campo
  • offline-first: guardar local siempre, sincronizar con sincronizado=0 cuando llega internet
  • onConnectivityChanged dispara la sincronización automáticamente al reconectar
