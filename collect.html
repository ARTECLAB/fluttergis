Antes de seguir
Por qué tocar el mapa y no solo usar el GPS del teléfono
📡
Solo GPS automático
La colecta queda donde el teléfono dice que estás — con el error normal del GPS (5-15 metros) y sin poder corregirlo si el poste, árbol o punto real está a unos metros de distancia.
👆
Tocar el mapa — lo que hacemos hoy
Ves tu posición GPS en vivo (Clase 2) como referencia, y tocas el punto exacto del elemento que quieres registrar. Más preciso, y reutiliza el mapa que ya construiste.
✓
Una sola app, todo conectado
Este es el corazón del curso: el mapa con GPS y capas de las Clases 1-4 no queda aislado — hoy lo conectamos con SQLite para que capturar datos de campo sea parte de la misma app, no una pantalla suelta.
📍
El GPS en vivo no desaparece: sigue mostrando tu posición mientras decides dónde tocar. Solo cambia de dónde sale el punto que se guarda.

⚠️
Sin CAMERA, la foto falla
ImagePicker().pickImage(source: ImageSource.camera) se cierra sola o lanza excepción si falta el permiso en el Manifest — no basta con pedirlo en runtime.
✓
¿Para qué es cada paquete?
sqflite+path: base de datos local. path_provider: encontrar dónde guardar el .db en cada plataforma. image_picker: tomar fotos. connectivity_plus: detectar cuándo hay internet para sincronizar.
El modelo de datos
La clase Colecta — toMap() y fromMap()
lib/models/colecta.dart
class Colecta { final int id; final String tipo; final String descripcion; final double latitud; final double longitud; final String? fotoPath; final int timestamp; final bool sincronizado; Map<String,dynamic> toMap() => { 'id': id, 'tipo': tipo, 'descripcion': descripcion, 'latitud': latitud, 'longitud': longitud, 'fotoPath': fotoPath, 'timestamp': timestamp, // bool no existe en SQLite 'sincronizado': sincronizado ? 1 : 0, }; factory Colecta.fromMap(Map m) => Colecta( id: m['id'], tipo: m['tipo'], descripcion: m['descripcion'], latitud: m['latitud'], longitud: m['longitud'], fotoPath: m['fotoPath'], timestamp: m['timestamp'], sincronizado: m['sincronizado'] == 1, ); }
⚠️
bool → INTEGER en SQLite
SQLite no tiene tipo BOOLEAN. La IA pasa el bool directamente y lanza error en runtime. Convertir siempre: sincronizado ? 1 : 0
✓
ID único sin servidor: DateTime.now().millisecondsSinceEpoch — garantiza unicidad sin autoincrement del servidor.
Base de datos SQLite
Inicializar, insertar y consultar colectas
lib/data/colecta_db.dart
class ColectaDB { static Database? _db; static Future<Database> get db async { _db ??= await _initDb(); return _db!; } static Future<Database> _initDb() async { final dir = await getApplicationDocumentsDirectory(); return openDatabase( join(dir.path, 'geocollect.db'), version: 1, onCreate: (db, _) => db.execute(''' CREATE TABLE colectas ( id INTEGER PRIMARY KEY, tipo TEXT NOT NULL, descripcion TEXT, latitud REAL NOT NULL, longitud REAL NOT NULL, fotoPath TEXT, timestamp INTEGER NOT NULL, sincronizado INTEGER DEFAULT 0)'''), ); } static Future<int> insertar(Colecta c) async => (await db).insert( 'colectas', c.toMap(), conflictAlgorithm: ConflictAlgorithm.replace); static Future<List<Colecta>> pendientes() async { final rows = await (await db).query( 'colectas', where: 'sincronizado = 0'); return rows.map(Colecta.fromMap).toList(); } }
📁
getApplicationDocumentsDirectory() devuelve la ruta correcta en cada plataforma — el código Dart es idéntico en Android e iOS.
✓
sincronizado = 0 en la consulta SQL filtra las colectas pendientes — base del sistema offline-first. Al sincronizar, actualizar a 1.

Confirmar y enviar el punto al formulario
lib/screens/mapa_screen.dart — navegar con el punto
void _confirmarColecta(LatLng punto) { showDialog( context: context, builder: (_) => AlertDialog( title: Text('¿Colectar este punto?'), content: Text( '${punto.latitude.toStringAsFixed(5)}, ' '${punto.longitude.toStringAsFixed(5)}'), actions: [ TextButton( onPressed: () => Navigator.pop(context), child: Text('Cancelar')), FilledButton( onPressed: () { Navigator.pop(context); // cierra el diálogo Navigator.push(context, MaterialPageRoute(builder: (_) => CollectScreen(punto: punto))); }, child: Text('Colectar aquí')), ], ), ); }
✓
Un diálogo antes de navegar
Evita colectas accidentales por un toque sin querer — el usuario confirma coordenadas antes de abrir el formulario. Cancelar simplemente cierra el diálogo, sin navegar a ningún lado.
➡️
CollectScreen(punto: punto)
El LatLng capturado viaja como parámetro del constructor — la pantalla de formulario nunca necesita pedirle nada al GPS, ya recibe el punto listo.
Offline-first
Guardar local + sincronizar cuando hay internet
Ciclo offline-first completo
// Al guardar la colecta: Future<void> guardarColecta( Colecta c) async { // Siempre guardar local primero await ColectaDB.insertar(c .copyWith(sincronizado: false)); // Intentar sincronizar si hay internet final conn = await Connectivity() .checkConnectivity(); if (conn != ConnectivityResult.none) { await _sincronizar(); } } // Escuchar reconexion automaticamente void _listenConectividad() { Connectivity().onConnectivityChanged .listen((result) { if (result != ConnectivityResult.none) { _sincronizar(); } }); } Future<void> _sincronizar() async { final pendientes = await ColectaDB.pendientes(); for (final c in pendientes) { // Publicar en GeoServer via REST await _publicarEnGeoServer(c); // Marcar como sincronizado await ColectaDB.marcarSincronizado(c.id); } }
✓
Flujo offline-first
1. Guardar en SQLite → 2. Intentar sincronizar → 3. Si no hay internet: esperar. 4. Al reconectar: sincronizar automáticamente. Nunca se pierde una colecta.
📡
onConnectivityChanged es un Stream que emite cuando cambia la red — el dispositivo detecta la reconexión automáticamente y dispara la sincronización.
Cómo se usa · Parte 1 de 3
CollectScreen — recibe el punto, ya no lo pide al GPS
lib/screens/collect_screen.dart — constructor y build()
class CollectScreen extends StatefulWidget { final LatLng punto; const CollectScreen({required this.punto, super.key}); @override State<CollectScreen> createState() => _CollectScreenState(); } class _CollectScreenState extends State<CollectScreen> { final _tipoCtrl = TextEditingController(); String? _fotoPath; // widget.punto: ya llegó desde el mapa Widget build(BuildContext ctx) { final p = widget.punto; return Scaffold( appBar: AppBar(title: Text('Nueva colecta')), body: Padding( padding: EdgeInsets.all(16), child: Column(children: [ TextField(controller: _tipoCtrl, decoration: InputDecoration( labelText: 'Tipo')), Text('📍 ${p.latitude.toStringAsFixed(5)}, ' '${p.longitude.toStringAsFixed(5)}'), ElevatedButton.icon( onPressed: _tomarFoto, icon: Icon(Icons.camera_alt), label: Text('Tomar foto')), ElevatedButton( onPressed: _guardar, child: Text('Guardar colecta')), ]), ), ); } }
🧩
widget.punto
Desde el State, widget te da acceso al StatefulWidget padre — así se lee el punto que llegó por el constructor. Nunca se vuelve a pedir posición al GPS aquí.
✓
El botón Guardar ya no se deshabilita
Como el punto llega listo desde el mapa, no hay que esperar ni mostrar un spinner — a diferencia del GPS automático, aquí no existe el estado "todavía no sé dónde estoy".

Cómo se usa · Parte 3 de 3
_guardar() — arma el Colecta con widget.punto y lo guarda offline-first
lib/screens/collect_screen.dart — _guardar()
Future<void> _guardar() async { final ahora = DateTime.now().millisecondsSinceEpoch; final c = Colecta( id: ahora, tipo: _tipoCtrl.text, descripcion: '', latitud: widget.punto.latitude, longitud: widget.punto.longitude, fotoPath: _fotoPath, timestamp: ahora, sincronizado: false, ); // Guarda local y sincroniza si hay red await guardarColecta(c); if (!mounted) return; Navigator.pop(ctx); // volver al mapa }
✓
sincronizado: false siempre al crear
La colecta nueva nunca nace sincronizada — eso lo decide guardarColecta() según si hay internet en ese momento, no la pantalla.
⚠️
if (!mounted) antes de Navigator.pop
await guardarColecta(c) es asíncrono — si el usuario cierra la pantalla mientras espera, ctx ya no es válido. mounted evita ese crash.
IA como copiloto
Generar CollectScreen con la IA
💬
"Genera un StatefulWidget CollectScreen en Flutter que reciba un LatLng 'punto' obligatorio por el constructor. Debe mostrar un campo de texto (tipo), el punto recibido, un botón para tomar foto con image_picker, y un botón Guardar que arme un objeto Colecta usando ese punto y llame a guardarColecta(). No debe pedirle nada al GPS — el punto ya llega listo."
Lo que la IA hace bien
El layout con Column, los TextField, el ElevatedButton.icon para la foto, y el parámetro requerido del constructor. Correcto al 80%.
Errores típicos de la IA
1) Agrega de todos modos un Geolocator.getCurrentPosition() "por si acaso" — duplica lo que ya resolvió el mapa. 2) Olvida if (!mounted) antes de usar context tras un await. 3) Pone sincronizado: true por defecto — rompe el offline-first.
🔍
Verificación rápida
Revisa el código generado: si aparece cualquier mención a Geolocator dentro de CollectScreen, sobra — esa pantalla solo debe leer widget.punto.
Resumen
Lo que aprendiste hoy
  • onTap en MapOptions captura el LatLng exacto donde el usuario toca el mapa
  • Navigator.push + CollectScreen(punto: punto) envía la coordenada al formulario — sin volver a pedirle nada al GPS
  • SQLite no tiene BOOLEAN — siempre convertir: sincronizado ? 1 : 0. La IA olvida esto
  • toMap() para insertar en SQLite, fromMap() para reconstruir desde el resultado SQL
  • imageQuality: 30 comprime de 3MB a ~80KB — multiplica x37 la capacidad de almacenamiento en campo
  • offline-first: guardar local siempre, sincronizar con sincronizado=0 cuando llega internet
  • onConnectivityChanged dispara la sincronización automáticamente al reconectar
