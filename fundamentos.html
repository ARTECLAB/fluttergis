Dart esencial

El curso usa Dart constantemente. No necesitas ser experto, pero sí debes sentirte cómodo con estas construcciones. Si alguna te resulta extraña, repásala antes.

💡
¿Por qué Dart?
Flutter está escrito en Dart. Es el lenguaje que usamos para construir la app GIS: el mapa, el GPS, la base de datos y la exportación a GeoJSON son todos Dart puro.
Repaso rápido — lo que usarás en el curso
// Variables y tipos String nombre = 'La Paz'; double lat = -16.5000; int zoom = 12; bool activo = true; var ciudad = 'Cochabamba'; // Dart infiere el tipo // Funciones double calcularDistancia(double lat1, double lon1) { return lat1 * lon1; // simplificado } // Future — operación asíncrona (resultado único) Future<String> obtenerDato() async { await Future.delayed(Duration(seconds: 1)); return 'Bolivia'; } // Stream — flujo continuo de valores (ej: GPS) Stream<double> temperaturaStream() async* { while (true) { yield 25.0; // emite un valor await Future.delayed(Duration(seconds: 1)); } } // List — la usamos para trayectorias GPS final puntos = <String>['A', 'B', 'C']; puntos.add('D'); puntos.where((p) => p != 'B').toList(); // Map — lo usamos para SQLite y GeoJSON final colecta = <String, dynamic>{ 'tipo': 'árbol', 'latitud': -16.50, 'timestamp': 1700000000, };
🔄
Future vs Stream
Future: un solo resultado. Como una foto. Stream: muchos resultados en el tiempo. Como un video. El GPS es un Stream.
⚡
async / await
Marcar una función como async permite usar await dentro. El código espera sin congelar la app.
📦
Map<String, dynamic>
El tipo de datos más frecuente del curso. SQLite devuelve Maps. GeoJSON es un Map. Aprende a leerlos y escribirlos.

Conceptos GIS esenciales

No necesitas ser geógrafo. Pero sí entender qué son coordenadas, qué diferencia hay entre WMS y WFS, y por qué GeoJSON usa [lon, lat] al revés de lo que parecería lógico.

📍
Coordenadas geográficas
Latitud: norte/sur (−90 a 90). Longitud: este/oeste (−180 a 180). La Paz: lat −16.50, lon −68.15. Dart usa siempre LatLng(lat, lon).
🔢
Niveles de zoom
Zoom 1 = planeta entero. Zoom 6 = país. Zoom 12 = ciudad. Zoom 16 = calle. Más zoom = más teselas = más detalle.
🖼️
WMS — imagen renderizada
El servidor GeoServer dibuja el mapa y devuelve un PNG. Rápido para mostrar, pero no puedes tocar ni leer los datos.
📦
WFS — datos vectoriales
GeoServer devuelve los datos crudos en GeoJSON. Puedes tocar cada polígono, leer atributos y analizarlos en la app.
🧩
Tiles / Teselas
El mapa se divide en imágenes de 256×256px llamadas teselas. flutter_map las descarga y ensambla automáticamente.
📄
GeoJSON
Formato estándar para datos geoespaciales en JSON. Un Feature tiene geometry (la forma) y properties (los atributos).
⚠️
El error más frecuente del curso
GeoJSON usa [longitud, latitud] — al revés de LatLng(latitud, longitud) de flutter_map. Si tus puntos aparecen en el océano, es este error. Recuérdalo: GeoJSON = [lon, lat].
Estructura de un GeoJSON — lo exportaremos en la Clase 9
{ "type": "FeatureCollection", "features": [ { "type": "Feature", "geometry": { "type": "Point", // ⚠ GeoJSON: [longitud, latitud] "coordinates": [-68.1500, -16.5000] }, "properties": { "tipo": "árbol", "especie": "Molle" } } ] }

Glosario

Términos clave del curso

Palabras que aparecerán desde la primera clase.

LatLng
LatLng(latitud, longitud)
La unidad básica de posición en flutter_map. Siempre lat primero, lon segundo.
OSM
OpenStreetMap
Mapa colaborativo y libre. Lo usamos como mapa base sin pagar ni registrarse.
WMS
Web Map Service
Estándar OGC. GeoServer devuelve una imagen PNG del mapa. Clases 3 del curso.
WFS
Web Feature Service
Estándar OGC. GeoServer devuelve datos vectoriales en GeoJSON. Clase 4.
FMTC
Flutter Map Tile Caching
Paquete para cachear y pre-descargar tiles. Permite usar el mapa sin internet. Clase 8.
CQL
Common Query Language
Filtros de GeoServer. Ejemplo: departamento='LA PAZ' para mostrar solo ese departamento.
