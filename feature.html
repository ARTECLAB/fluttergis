¿Por qué GeoJSON local?
Assets en la app vs petición a un servidor
📁
GeoJSON en assets/ — lo que usaremos
El archivo .geojson va dentro de la app. Carga instantánea, funciona offline, no depende de que el servidor esté encendido. Ideal para datos que cambian poco: provincias, departamentos.
🌐
Petición a un servidor
Requiere conexión, el servidor debe estar activo y puede haber problemas de CORS en algunos entornos. Útil para datos que cambian frecuentemente.
✓
En el curso usamos assets/
Los límites de provincias y departamentos de Bolivia no cambian a diario. Tenerlos en la app garantiza que la Clase 4 funciona sin importar la conectividad del aula.
📥
El instructor provee el archivo provincias_bolivia.geojson — el mapa de las 112 provincias de Bolivia. Se coloca en assets/geo/ y se declara en pubspec.yaml.
Configuración
Registrar el GeoJSON en pubspec.yaml y leerlo
pubspec.yaml — declarar el asset
flutter: assets: - assets/geo/provincias_bolivia.geojson
Leer el archivo desde la app
import 'package:flutter/services.dart'; import 'dart:convert'; // Leer el GeoJSON como String Future<List<Map>> cargarProvincias() async { final json = await rootBundle.loadString( 'assets/geo/provincias_bolivia.geojson' ); // compute() parsea en Isolate separado // para no congelar el mapa return compute(_parsear, json); } // Esta función corre en el Isolate List<Map> _parsear(String json) { final data = jsonDecode(json); return List<Map>.from( data['features'] as List); }
✓
rootBundle.loadString()
Lee cualquier archivo de assets/ como texto. No requiere permisos, no necesita internet. Es síncrono en lectura pero lo envolvemos en async para no bloquear.
⚠️
compute() es obligatorio aquí
Parsear 112 provincias puede tardar 100-300ms y congelar el mapa. compute() corre esa función en un Isolate — un hilo aislado con su propia memoria — y la UI sigue fluida.
Error crítico
GeoJSON: [lon, lat]
LatLng: (lat, lon)
El error más frecuente al mostrar datos geoespaciales en Flutter
Error más frecuente
El orden de coordenadas — siempre al revés del que esperas
Conversión correcta de GeoJSON a Polygon
List<Polygon> buildPolygons( List<Map> features) { return features.map((f) { final coords = f['geometry']['coordinates'][0] as List; final points = coords.map((c) => LatLng( // GeoJSON: c[0]=lon, c[1]=lat // LatLng: (lat, lon) — al revés (c[1] as num).toDouble(), // lat (c[0] as num).toDouble(), // lon )).toList(); return Polygon( points: points, color: Color(0xFF006A6A) .withOpacity(0.25), borderColor: Color(0xFF006A6A), borderStrokeWidth: 1.5, ); }).toList(); }
🚨
El error más cometido
Poner c[0] como latitud. Las provincias aparecen en el océano Pacífico o en África. Siempre: c[1] = lat, c[0] = lon en GeoJSON.
🎨
Las 3 propiedades visuales de Polygon
color es el relleno (con .withOpacity(0.25) para verlo semitransparente y no tapar el mapa base). borderColor es el color de la línea del borde. borderStrokeWidth es el grosor de esa línea en píxeles.
PolygonLayer
Dibujar el mapa completo de provincias
FlutterMap con PolygonLayer — build() completo
Widget build(BuildContext ctx) { return FlutterMap( options: MapOptions( initialCenter: LatLng(-16.50, -64.50), initialZoom: 5, ), children: [ // 1. Mapa base, siempre primero TileLayer( urlTemplate: 'https://tile.openstreetmap' '.org/{z}/{x}/{y}.png', userAgentPackageName: 'com.arteclab.geocollect', ), // 2. Las 112 provincias, encima del mapa PolygonLayer( polygons: _polygons, ), ], ); }
🗺️
¿Qué es PolygonLayer?
Es el widget de flutter_map que dibuja una lista de Polygon sobre el mapa — cada uno con sus propios puntos, color de relleno y borde. Aquí _polygons es el resultado de buildPolygons() de la diapositiva anterior.
✓
El orden en children importa
TileLayer va primero porque es la base. Cualquier capa después se dibuja encima — por eso PolygonLayer va después, o las provincias quedarían tapadas por el mapa base.
IA como copiloto
Lo que la IA genera bien y lo que olvida
💬
"Genera una función Dart que carga un GeoJSON de provincias desde rootBundle, lo parsea con compute() y devuelve List con PolygonLayer de flutter_map 8.x. Con colores teal."
Lo que la IA hace bien
La estructura de rootBundle.loadString(), jsonDecode, compute() y la construcción de Polygon. Correcto al 80%.
Lo que la IA olvida
El orden lon/lat: la IA frecuentemente pone c[0] como latitud. También suele olvidar declarar el asset en pubspec.yaml.
🔍
Verificación inmediata después de generar
Correr la app y verificar que las provincias aparecen sobre Bolivia — no en el océano Pacífico. Si están en el lugar equivocado: buscar en el código generado si c[0] y c[1] están invertidos.
Resumen
Lo que aprendiste hoy
  • GeoJSON en assets/ funciona offline desde el primer día — sin servidores, sin CORS, sin problemas de red
  • Declarar el asset en pubspec.yaml y leerlo con rootBundle.loadString()
  • compute() parsea el GeoJSON en un Isolate separado — el mapa no se congela
  • GeoJSON usa [longitud, latitud] — LatLng quiere (latitud, longitud). c[1]=lat, c[0]=lon. Siempre
  • Polygon se dibuja con color (relleno), borderColor y borderStrokeWidth (borde)
  • PolygonLayer dibuja todas las provincias de una vez, encima del TileLayer base
