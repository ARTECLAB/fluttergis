WMS vs WFS
¿Cuándo usar cada uno?
🖼️
WMS — imagen renderizada
GeoServer dibuja el mapa y envía un PNG. Rápido para mostrar, pero el usuario no puede tocar ni leer atributos de features individuales.
📦
WFS — datos crudos
GeoServer envía el GeoJSON con geometrías y atributos. Se puede tocar un municipio y leer nombre, población, área, código INE.
✓
Regla práctica
WMS para fondos y capas decorativas. WFS cuando el usuario necesita interactuar y leer atributos de cada feature.
🔗
GeoServer 3 sirve la misma capa como WMS y WFS simultáneamente. La app puede usar los dos para la misma capa con propósitos distintos.
Código esencial
Petición WFS + compute() para no congelar el mapa
Descargar GeoJSON de municipios
Future<List<Map>> fetchMunicipios() async { const url = 'http://servidor:8080/geoserver/wfs' '?SERVICE=WFS&VERSION=2.0.0' '&REQUEST=GetFeature' '&TYPENAMES=geoserver3_curso:municipios' '&OUTPUTFORMAT=application/json' '&COUNT=50' "&CQL_FILTER=departamento='LA PAZ'"; final resp = await http.get(Uri.parse(url)); // compute() parsea en Isolate separado // para no congelar el mapa (200ms+) return compute(_parse, resp.body); } // Esta funcion corre en el Isolate List<Map> _parse(String body) { final data = jsonDecode(body); return List<Map>.from(data['features']); }
⚠️
compute() es obligatorio aquí
Sin compute(), parsear 336 municipios puede tardar 200ms y congelar el mapa visiblemente. La IA suele olvidar esto.
📌
COUNT=50 limita la descarga a 50 features. Sin esto, WFS devuelve todos los municipios del país — miles de polígonos que saturan la app.
Error crítico
GeoJSON: [lon, lat]
LatLng: (lat, lon)
El error #1 de todos los desarrolladores GIS en Flutter
Error más frecuente
El orden de coordenadas en GeoJSON vs flutter_map
Conversión correcta de GeoJSON a Polygon
List<Polygon> buildPolygons(List<Map> features) { return features.map((f) { final coords = f['geometry'] ['coordinates'][0] as List; final points = coords.map((c) => LatLng( // GeoJSON: c[0]=lon, c[1]=lat // LatLng: (lat, lon) (c[1] as num).toDouble(), // lat primero (c[0] as num).toDouble(), // lon segundo )).toList(); return Polygon( points: points, color: Color(0xFF006A6A).withOpacity(0.3), borderColor: Color(0xFF006A6A), borderStrokeWidth: 1.5, hitValue: f['properties'], ); }).toList(); }
🚨
El error más cometido
Poner c[0] como latitud y c[1] como longitud. El mapa muestra los municipios en el océano Pacífico o en África. Siempre: c[1]=lat, c[0]=lon en GeoJSON.
✓
hitValue en Polygon pasa los atributos del municipio al callback de tap — nombre, población, código INE disponibles al tocar.
Hack
Cache GeoJSON en SQLite:
offline desde el segundo clic
TTL de 24h — los límites municipales no cambian a diario
Hack bueno de la clase
Cache inteligente con TTL de 24 horas
Leer cache o descargar
Future<String> getGeojsonCacheado( String key, String url) async { final db = await openDatabase('cache.db'); final rows = await db.query('cache', where: 'key = ?', whereArgs: [key]); if (rows.isNotEmpty) { final edad = DateTime.now() .millisecondsSinceEpoch - (rows.first['ts'] as int); // 86400000 ms = 24 horas if (edad < 86400000) return rows.first['data'] as String; } // Cache expirado o vacío: descargar final resp = await http.get(Uri.parse(url)); await db.insert('cache', { 'key': key, 'data': resp.body, 'ts': DateTime.now().millisecondsSinceEpoch, }, conflictAlgorithm: ConflictAlgorithm.replace); return resp.body; }
Primera cargaDescarga WFS (~1-3 segundos)
Siguientes 24hSQLite local (~10ms)
Sin internetCache aunque haya expirado
✓
Los límites de los municipios de Bolivia cambian cada años. Un TTL de 24h es más que conservador — los datos siempre son precisos.
IA como copiloto
Prompt para generar el parser WFS
💬
"Escribe una función Dart async que pide un WFS GeoJSON a una URL y retorna List con el centroide de cada polígono. Incluye manejo de errores y usa compute() para el parseo."
Lo que la IA hace bien
El HTTP request, jsonDecode, el manejo básico de errores y la estructura del Future. Generalmente correcto al 80%.
Lo que la IA olvida
Añadir compute() para el parseo, y con frecuencia invierte el orden lat/lon al construir LatLng desde el GeoJSON.
🔍
Verificación inmediata
Después de generar el código, verificar que: 1) usa compute(), 2) el LatLng tiene c[1] como latitud y c[0] como longitud. Estos son los dos errores más frecuentes del código generado.
Resumen
Lo que aprendiste hoy
  • WFS devuelve GeoJSON con atributos interactivos vs WMS que solo muestra imágenes
  • compute(funcion, body) parsea el JSON en Isolate separado sin congelar la UI
  • GeoJSON usa [longitud, latitud] — LatLng quiere (latitud, longitud). c[1]=lat, c[0]=lon
  • PolygonLayer + hitValue permite tocar cada polígono y leer sus atributos
  • Cache SQLite con TTL de 24h: segunda carga instantánea y offline-ready
