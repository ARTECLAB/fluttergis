¿Por qué GeoJSON local?
Assets en la app vs petición a un servidor
📁
GeoJSON en assets/ — lo que usaremos
El archivo .geojson va dentro de la app. Carga instantánea, funciona offline, no depende de que el servidor esté encendido. Ideal para datos que cambian poco: municipios, departamentos.
🌐
Petición a un servidor
Requiere conexión, el servidor debe estar activo y puede haber problemas de CORS en algunos entornos. Útil para datos que cambian frecuentemente.
✓
En el curso usamos assets/
Los límites de municipios y departamentos de Bolivia no cambian a diario. Tenerlos en la app garantiza que la Clase 4 funciona sin importar la conectividad del aula.
📥
El instructor provee el archivo municipios_bolivia.geojson. Se coloca en assets/geo/ y se declara en pubspec.yaml.
Configuración
Registrar el GeoJSON en pubspec.yaml y leerlo
pubspec.yaml — declarar el asset
flutter: assets: - assets/geo/municipios_bolivia.geojson
Leer el archivo desde la app
import 'package:flutter/services.dart'; import 'dart:convert'; // Leer el GeoJSON como String Future<List<Map>> cargarMunicipios() async { final json = await rootBundle.loadString( 'assets/geo/municipios_bolivia.geojson' ); // compute() parsea en Isolate separado // para no congelar el mapa return compute(_parsear, json); } // Esta función corre en el Isolate List<Map> _parsear(String json) { final data = jsonDecode(json); return List<Map>.from( data['features'] as List); }
✓
rootBundle.loadString()
Lee cualquier archivo de assets/ como texto. No requiere permisos, no necesita internet. Es síncrono en lectura pero lo envolvemos en async para no bloquear.
⚠️
compute() es obligatorio aquí
Parsear 336 municipios puede tardar 100-300ms. Sin compute() el mapa se congela visiblemente. Con compute() corre en un Isolate y la UI sigue fluida.
Error crítico
GeoJSON: [lon, lat]
LatLng: (lat, lon)
El error más frecuente al mostrar datos geoespaciales en Flutter
Error más frecuente
El orden de coordenadas — siempre al revés del que esperas
Conversión correcta de GeoJSON a Polygon
List<Polygon> buildPolygons( List<Map> features) { return features.map((f) { final coords = f['geometry']['coordinates'][0] as List; final points = coords.map((c) => LatLng( // GeoJSON: c[0]=lon, c[1]=lat // LatLng: (lat, lon) — al revés (c[1] as num).toDouble(), // lat (c[0] as num).toDouble(), // lon )).toList(); return Polygon( points: points, color: Color(0xFF006A6A) .withOpacity(0.25), borderColor: Color(0xFF006A6A), borderStrokeWidth: 1.5, // hitValue pasa los atributos al tap hitValue: f['properties'], ); }).toList(); }
🚨
El error más cometido
Poner c[0] como latitud. Los municipios aparecen en el océano Pacífico o en África. Siempre: c[1] = lat, c[0] = lon en GeoJSON.
✓
hitValue en Polygon pasa las propiedades del municipio al callback de tap — nombre, departamento, código INE, todo disponible al tocar.
PolygonLayer
Polígonos interactivos sobre el mapa
FlutterMap con PolygonLayer + popup al tocar
final _hitNotifier = ValueNotifier<LayerHitResult?>(null); FlutterMap( children: [ TileLayer(urlTemplate: _urlOSM), PolygonLayer( polygons: _polygons, hitNotifier: _hitNotifier, ), // Detectar tap en polígono GestureDetector( onTap: () { final hit = _hitNotifier.value; if (hit == null) return; final props = hit.hitValues.first as Map<String, dynamic>; _mostrarPopup( props['nombre'], props['departamento'], ); }, child: Container(color: Colors.transparent), ), ], )
💡
hitNotifier + GestureDetector
Es el mecanismo de flutter_map 8.x para detectar tap en polígonos. El GestureDetector transparente cubre todo el mapa y el hitNotifier indica qué polígono se tocó.
✓
hit.hitValues.first devuelve el hitValue que pusimos en el Polygon — que son las propiedades del municipio del GeoJSON.

offline desde el segundo uso

Hack bueno de la clase
Guardar los features parseados en SQLite
Cache del GeoJSON parseado
Future<List<Map>> getFeatures() async { final db = await ColectaDb.db; // ¿Ya hay datos en caché? final cached = await db.query( 'geo_cache', where: 'key = ?', whereArgs: ['municipios'], ); if (cached.isNotEmpty) { // Devolver desde SQLite (~5ms) return (jsonDecode( cached.first['data'] as String) as List).cast<Map>(); } // Primera vez: cargar desde assets final features = await cargarMunicipios(); // Guardar en SQLite para la próxima await db.insert('geo_cache', { 'key': 'municipios', 'data': jsonEncode(features), }); return features; }

IA como copiloto
Lo que la IA genera bien y lo que olvida
💬
"Genera una función Dart que carga un GeoJSON de municipios desde rootBundle, lo parsea con compute() y devuelve List con PolygonLayer de flutter_map 8.x. Con colores teal."
Lo que la IA hace bien
La estructura de rootBundle.loadString(), jsonDecode, compute() y la construcción de Polygon. Correcto al 80%.
Lo que la IA olvida
El orden lon/lat: la IA frecuentemente pone c[0] como latitud. También suele olvidar declarar el asset en pubspec.yaml.
🔍
Verificación inmediata después de generar
Correr la app y verificar que los municipios aparecen sobre Bolivia — no en el océano Pacífico. Si están en el lugar equivocado: buscar en el código generado si c[0] y c[1] están invertidos.
Resumen
Lo que aprendiste hoy
  • GeoJSON en assets/ funciona offline desde el primer día — sin servidores, sin CORS, sin problemas de red
  • Declarar el asset en pubspec.yaml y leerlo con rootBundle.loadString()
  • compute() parsea el GeoJSON en un Isolate separado — el mapa no se congela
  • GeoJSON usa [longitud, latitud] — LatLng quiere (latitud, longitud). c[1]=lat, c[0]=lon. Siempre
  • PolygonLayer con hitValue + GestureDetector — detectar tap y mostrar atributos del municipio
  • Cache en SQLite: primera carga desde assets (~200ms), siguientes desde base de datos (~5ms)
