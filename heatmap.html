Por que heatmap
El problema que resuelve — antes y después

500 marcadores individuales
El mapa se llena de pines solapados. La densidad de eventos es invisible. El usuario no puede distinguir nada.

Heatmap de los mismos 500 puntos
Las zonas de alta concentración aparecen rojas, las de baja densidad azules. El patrón espacial es inmediatamente evidente.

Normalización min-max y filtro de BBox visible
buildHeatmap — el metodo clave
import 'dart:math' as math; List<WeightedLatLng> buildHeatmap( List<Map> puntos, LatLngBounds bounds) { // 1. Solo puntos en el area visible final vis = puntos.where((p) => bounds.contains( LatLng(p['lat'], p['lon']))) .toList(); if (vis.isEmpty) return []; // 2. Normalizacion min-max final vals = vis.map( (p) => p['intensidad'] as double).toList(); final mn = vals.reduce(math.min); final mx = vals.reduce(math.max); final rango = mx - mn; // 3. Construir WeightedLatLng // (constructor posicional: latLng, intensity — // sin nombre de parametro) return vis.map((p) => WeightedLatLng( LatLng(p['lat'], p['lon']), rango == 0 ? 0.5 : (p['intensidad'] - mn) / rango, )).toList(); }
⚡
BBox filter = rendimiento
Con 10,000 puntos en Bolivia y el mapa mostrando La Paz, filtrar por bounds reduce de 10,000 a ~500 puntos — el heatmap se calcula 20x mas rapido.
📐
Obtener el BBox actual: mapController.camera.visibleBounds — devuelve los limites norte, sur, este, oeste del area visible.
Codigo completo
HeatMapLayer en FlutterMap con Slider de radio
Integracion en FlutterMap
import 'dart:async'; double _radio = 30; final _ctrl = MapController(); final _reset = StreamController<void>.broadcast(); FlutterMap( mapController: _ctrl, children: [ TileLayer(urlTemplate: _urlOSM), HeatMapLayer( heatMapDataSource: InMemoryHeatMapDataSource( data: buildHeatmap( _puntos, _ctrl.camera.visibleBounds), ), heatMapOptions: HeatMapOptions(radius: _radio), reset: _reset.stream, ), ], ), Slider( value: _radio, min: 10, max: 80, onChanged: (v) { setState(() => _radio = v); _reset.add(null); }, )
⚠️
setState() solo no repinta el heatmap
HeatMapLayer dibuja sus manchas de calor como teselas de un TileLayer interno, cacheadas por URL. Esa URL solo cambia cuando el stream reset emite un evento — por eso hace falta _reset.add(null) cada vez que cambian los datos o las opciones.

Resumen
Lo que aprendiste hoy
  • Heatmap resuelve el solapamiento de marcadores mostrando densidad como gradiente continuo
  • WeightedLatLng(latLng, intensity) — constructor posicional, cada punto contribuye segun su peso normalizado de 0.0 a 1.0
  • Normalizacion min-max: (valor - min) / (max - min) mapea cualquier rango a 0.0-1.0
  • Filtrar por visibleBounds reduce el dataset de miles a cientos — rendimiento critico
  • mapController.camera.visibleBounds da los limites del area visible en pantalla en tiempo real
  • Slider + setState() cambia el radio, pero solo se ve reflejado al emitir un evento por el stream reset — cerrarlo en dispose()
