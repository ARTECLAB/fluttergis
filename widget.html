Hack #1
Dark map automático:
el mapa que sabe el tema
CartoDB Dark Matter cuando el SO está en modo oscuro — sin que el usuario configure nada
Hack bueno de la clase
Cambio automático de mapa base según el tema del sistema
Detección de tema + swap de URL
class MapaScreen extends StatelessWidget { // URLs de los dos mapas base static const _urlOSM = 'https://tile.openstreetmap.org/{z}/{x}/{y}.png'; static const _urlDark = 'https://{s}.basemaps.cartocdn.com' '/dark_all/{z}/{x}/{y}.png'; @override Widget build(BuildContext ctx) { // Detectar tema del sistema operativo final isDark = MediaQuery.of(ctx) .platformBrightness == Brightness.dark; return FlutterMap( options: MapOptions( initialCenter: LatLng(-16.50, -68.15), initialZoom: 12, ), children: [ TileLayer( // Swap automático sin setState urlTemplate: isDark ? _urlDark : _urlOSM, subdomains: isDark ? const ['a','b','c','d'] : const [], userAgentPackageName: 'com.arteclab.geocollect', ), ], ); } }
⚡
Por qué esto es elegante
MediaQuery.of(ctx) escucha cambios del sistema. Cuando el usuario activa modo oscuro, Flutter reconstruye la pantalla y el mapa cambia automáticamente. Cero lógica adicional.
🌐
Subdominios de CartoDB
Los subdominios a, b, c, d permiten paralizar la descarga de tiles usando 4 servidores simultáneos — el mapa carga 4x más rápido.

Lo que la IA hace bien
La estructura del Marker, el Container con BoxDecoration y el Tooltip están perfectos.
Lo que la IA olvida

Resumen
Lo que aprendiste hoy
  • flutter_map no requiere API key y soporta WMS/GeoServer nativamente — por eso lo elegimos
  • TileLayer con urlTemplate acepta cualquier proveedor de tiles XYZ: OSM, CartoDB, satélite, etc.
  • LatLng(lat, lon) — siempre latitud primero, longitud segundo
  • MarkerLayer con alignment: Alignment.topCenter posiciona el marcador correctamente sobre el punto
  • MediaQuery.platformBrightness detecta el tema del SO para cambiar el mapa automáticamente
  • La IA genera la estructura correcta pero olvida detalles como el alignment del marcador
