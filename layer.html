Un curso, dos aplicaciones
El curso GeoServer 3 publica las capas. Este curso (GeoCollect) las consume. Es el flujo de trabajo real de un proyecto GIS profesional.
🏷️
Notación workspace:capa
geoserver3_curso:departamentos sigue el formato workspace:nombre_capa — el workspace es el espacio de nombres de GeoServer.
Código esencial
TileLayer WMS — la forma correcta
TileLayer con WMSTileLayerOptions
TileLayer( wmsOptions: WMSTileLayerOptions( baseUrl: 'http://servidor:8080' '/geoserver/wms?', layers: ['geoserver3_curso:departamentos'], format: 'image/png', transparent: true, version: '1.3.0', // Filtro CQL dinámico otherParameters: { if (_filtro.isNotEmpty) 'CQL_FILTER': "nombre='$_filtro'", }, ), userAgentPackageName: 'com.arteclab.geocollect', // ⚠ NO añadir headers ni crossOrigin // flutter_map carga tiles como Image.network )
🔎
¿Qué es CQL_FILTER?
El lenguaje de filtro de GeoServer. Le pide al servidor que solo dibuje las entidades donde un atributo coincida — el filtrado ocurre en el servidor, no en la app.
🚨
El error más frecuente
Añadir additionalOptions: {'crossOrigin': 'anonymous'} o headers CORS. En apps móviles Flutter las teselas van como Image.network — CORS no aplica. Esto rompe la carga silenciosamente.
✓
transparent: true activa TRANSPARENT=true en la petición WMS. Combinar siempre con format: 'image/png' — JPEG no tiene transparencia.

Control de capas en FlutterMap — dentro de children:
// Estado por capa (variables de la pantalla) bool _showDptos = true; double _opDptos = 0.8; bool _showSatelital = false; // En FlutterMap children: Visibility( visible: _showDptos, child: Opacity( opacity: _opDptos, child: TileLayer( wmsOptions: WMSTileLayerOptions( layers: ['geoserver3_curso:departamentos'], // ... ), ), ), ), Visibility( visible: _showSatelital, child: TileLayer(/* WMS de la capa satelital (ráster) */), ),
💡
Visibility vs Opacity(0.0)
Visibility(visible: false) elimina el widget del árbol — no se renderiza ni consume recursos. Opacity(opacity: 0.0) renderiza el tile invisible, desperdiciando CPU y red.
✓
La opacidad es especialmente útil en capas ráster como satelital: a menos opacidad, más se ve el mapa base debajo de la imagen.

Widgets de control — SwitchListTile y Slider
// Panel de control (fuera del mapa): SwitchListTile( title: Text('Departamentos'), value: _showDptos, onChanged: (v) => setState(() => _showDptos = v), ), Slider( value: _opDptos, onChanged: (v) => setState(() => _opDptos = v), )
🎛️
SwitchListTile y Slider
Son widgets estándar de Flutter (Material), no específicos de GIS: SwitchListTile es un interruptor con etiqueta, Slider es una barra deslizante. Ambos solo necesitan value y onChanged.
✓
El slider de opacidad solo reconstruye el Opacity() wrapper — el TileLayer subyacente no se destruye ni recarga los tiles.
IA como copiloto
La IA genera el TileLayer WMS — y el error que cometería
Prompt exacto
💬
"Genera el TileLayer de flutter_map 8.x para consumir un WMS de GeoServer 3. URL base: http://servidor:8080/geoserver/wms. Workspace: geoserver3_curso, capa: departamentos. Con TRANSPARENT=true y FORMAT=image/png."
Lo que la IA hace bien
La estructura WMSTileLayerOptions, baseUrl, layers, format y transparent. Correcto al 90%.
Lo que la IA suele añadir de más
additionalOptions con crossOrigin o headers de autenticación innecesarios. Esto rompe la carga de tiles en apps móviles.
🌐
EPSG:4326 y BBOX
EPSG:4326 es el sistema de coordenadas (lat/lon, el mismo que GPS). BBOX es el área a dibujar — ojo: en WMS 1.3.0 con EPSG:4326 el orden es minLat,minLon,maxLat,maxLon, al revés de lo esperado.
🔍
Cómo detectar el error
Si los tiles del WMS no cargan pero la URL funciona en el navegador, busca en el código generado si hay crossOrigin, headers, o additionalOptions. Elimínalos.
Diagnóstico rápido
// Probar la URL directamente // en el navegador del simulador: http://servidor:8080/geoserver/wms ?SERVICE=WMS&VERSION=1.3.0 &REQUEST=GetMap &LAYERS=geoserver3_curso:departamentos &CRS=EPSG:4326 &BBOX=-22.9,-69.6,-9.7,-57.5 &WIDTH=512&HEIGHT=512 &FORMAT=image/png &TRANSPARENT=true
Resumen
Lo que aprendiste hoy
  • TileLayer + WMSTileLayerOptions conecta flutter_map a cualquier servidor WMS sin código adicional
  • NO añadir crossOrigin ni headers CORS — las teselas van como Image.network en apps móviles
  • Visibility(visible: bool) es más eficiente que Opacity(opacity: 0.0) para ocultar capas
  • Opacity() envuelve el TileLayer sin destruirlo — el slider funciona sin recargar tiles
  • otherParameters en WMSTileLayerOptions pasa CQL_FILTER y cualquier parámetro WMS adicional
  • Las capas de GeoServer 3 y Flutter GIS son dos cursos que trabajan juntos en producción real
